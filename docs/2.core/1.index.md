# Core

> Portable building blocks for agent tooling, plus small utilities.

The `unagent` package exports focused modules so you can import only what you need. Core modules are provider-agnostic primitives. Features are runtime integrations for specific environments or providers.

## Core Modules

| Module | Import | Description |
|--------|--------|-------------|
| [context](/core/context) | `unagent/context` | Token estimation, truncation, context building |
| [env](/core/env) | `unagent/env` | Agent detection, config paths, XDG |
| [exec](/core/exec) | `unagent/exec` | Safe command execution with timeout |
| [fs](/core/fs) | `unagent/fs` | Virtual filesystem (in-memory, overlay, real) |
| [git](/core/git) | `unagent/git` | Clone repos, git operations |
| [hooks](/core/hooks) | `unagent/hooks` | Agent lifecycle hooks |
| [link](/core/link) | `unagent/link` | Copy directories, symlinks |
| [lock](/core/lock) | `unagent/lock` | Skill lockfile management |
| [registry](/core/registry) | `unagent/registry` | Provider registry with priority matching |
| [skill](/core/skill) | `unagent/skill` | Skill discovery, parsing, install/uninstall |
| [source](/core/source) | `unagent/source` | URL and source string parsing |
| [stop](/core/stop) | `unagent/stop` | Stop conditions for agent loops |
| [stream](/core/stream) | `unagent/stream` | Stream handling, SSE formatting/parsing |
| [tool](/core/tool) | `unagent/tool` | Define tools, convert to JSON Schema |
| [usage](/core/usage) | `unagent/usage` | Token usage tracking, cost calculation |

## Full Import

The main entry point exports all public APIs from core modules and features.

```ts
import { detectCurrentAgent, discoverSkills, parseSource } from 'unagent'
```

## Selective Import

Individual module imports reduce bundle size by including only the code you need.

```ts
import { detectCurrentAgent } from 'unagent/env'
import { discoverSkills } from 'unagent/skill'
import { parseSource } from 'unagent/source'
```
