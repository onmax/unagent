# Jobs

> Durable background job dispatch and execution with provider-native handlers.

## Usage

```ts
import { createJobs } from 'unagent/jobs'
```

## Quick Start

```ts
const jobs = await createJobs({
  provider: {
    name: 'netlify',
    event: 'emails.send',
  },
  jobs: {
    'email:send': {
      async run(event) {
        return { result: { sent: true, to: event.payload.to } }
      },
    },
  },
})

await jobs.enqueue('email:send', { to: 'hello@example.com' }, { delaySeconds: 5 })
```

## Task vs Jobs

- `task` runs work directly (or from cron triggers) in the current runtime process.
- `jobs` enqueues durable background work and executes it from provider handlers.

Use `task` for local/scheduled execution and `jobs` for queue-backed background processing.

## API

### `createJobs(options)`

Create a jobs client with named handlers.

**Options:**
- `jobs` — `Record<string, JobEntry>` of named job handlers (plain `Job` objects or lazy `{ resolve, meta? }` entries)
- `provider` — provider options (Netlify in MVP)

**Returns `JobsClient`:**
- `enqueue(name, payload, options?)` — enqueue a named job
- `run(name, options?)` — run a job directly (uses task-runner semantics)
- `listJobs()` — list registered jobs and metadata
- `provider` — active provider name
- `supports` — capability flags
