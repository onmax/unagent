# Vercel

> Read-only access to Vercel project cron jobs. Schedules are configured at deploy time via `vercel.json`.

## Setup

```ts
import { createCron } from 'unagent/cron'

const cron = await createCron({
  provider: {
    name: 'vercel',
    token: process.env.VERCEL_TOKEN!,
    projectId: process.env.VERCEL_PROJECT_ID!,
  },
})
```

## List Schedules

```ts
const schedules = await cron.list()
```

## Get Schedule

```ts
const schedule = await cron.get('cron_123')
```

## Build Helper

Generate cron entries for `vercel.json`:

```ts
import { defineCronSchedules, toVercelCrons } from 'unagent/cron'

const schedules = defineCronSchedules([
  { cron: '*/5 * * * *', path: '/api/cleanup' },
  { cron: '0 0 * * *', path: '/api/daily-report' },
])

const vercelCrons = toVercelCrons(schedules)
// [{ path: '/api/cleanup', schedule: '*/5 * * * *' }, ...]
```

## Limitations

- Read-only at runtime (create/delete/pause/resume not supported)
- Schedules must be configured via `vercel.json` and deployed

## Capabilities

```ts
cron.supports
// { create: false, list: true, get: true, delete: false, pause: false, resume: false }
```
